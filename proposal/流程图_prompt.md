项目名称：HARM（Hey Asshole, Return Money）—— 一个 Bittensor 子网，用去中心化网络检测电商退款欺诈中的假图。

核心机制：商家提交可疑退款图片 → Validator 分发给多个 Miner → Miner 用 ML 模型或人工判断图片真假 → Validator 评分 → 链上分配奖励。

第 1 页：整体系统架构图

参与方：
- Bittensor 网络（Yuma 共识 + Emission 分配）
- Validator 节点（造假引擎 + 任务分发 + 评分器）
- Miner 节点（ML 模型 Miner / 人工 Miner / 混合 Miner）
- 商家客户（通过 API 提交可疑图片）

流向：
商家 → API → Validator → 分发任务给 Miner → Miner 返回检测结果 → Validator 评分 → 提交权重到 Bittensor 链 → Emission 分配给 Miner 和 Validator

第 2 页：Validator 完整工作流程

步骤：
1. 图片来源管理：70% 真实商家查询 + 30% 探针任务（自造假图）
2. 造假引擎（Forge Engine）：从干净图片生成假图，4 种篡改方法：
   - Copy-Move（复制粘贴图内区域）
   - Splicing（从其他图片拼接区域）
   - 压缩不一致（局部用不同 JPEG 质量重压缩）
   - 噪声注入（局部添加高斯噪声）
   每张探针图都记录：篡改方法 + ground truth mask
3. 任务分发：将探针和真实任务混合，下发给所有 Miner
4. 收集 Miner 响应
5. 评分（探针用 ground truth 评、真实任务用加权投票评）
6. 提交权重到 Bittensor 链

---

第 3 页：Miner 完整工作流程

步骤：
1. 从 Validator 接收待检测图片
2. 选择检测方式：
   - ML 模型路线：从 10 个模型后端中选择（Mesorch / TruFor / FOCAL / IML-ViT / ProFact / PSCC-Net 等）
   - 人工路线：肉眼观察 + 手动标注
   - 混合路线：模型初筛 + 人工复检
3. 执行检测分析
4. 生成响应：verdict（真/假）+ confidence（置信度 0-1）+ mask（篡改区域，可选）+ method（篡改方法，可选）
5. 通过 Commit-Reveal 协议提交结果

---

第 4 页：Commit-Reveal 防抄袭协议

时间线：
T0 — Validator 下发图片给所有 Miner
T1 — 每个 Miner 独立计算结果，提交 hash(结果 + 随机数)（Phase 1：承诺阶段）
T2 — 所有 Miner 的 hash 收齐
T3 — Miner 揭示原始结果 + 随机数（Phase 2：揭示阶段）
T4 — Validator 验证 hash 一致性 → 不一致者直接判零分 → 评分

关键点：Miner 在提交 hash 之后、揭示之前，无法看到其他人的答案，因此无法抄袭。

第 5 页：评分流程

四个评分维度：

A. 探针任务评分（权重 60%）— 有 ground truth：
   门控：verdict 判断正确 → 继续评分；判断错误 → 直接 0 分，并记录一次 Strike
   通过门控后：
   - 置信度校准（70%）：C_cal = 1 - |confidence - 理想值|
   - 篡改区域 IoU（20%，可选加分）：预测 mask 与真实 mask 的交并比
   - 方法识别（10%，可选加分）：正确识别篡改方法得 1 分

B. 共识任务评分（权重 35%）— 无 ground truth：
   加权多数投票，每个 Miner 的投票权重 = 其历史探针准确率
   与加权多数一致 → 得 1 分；不一致 → 得 0 分

C. 速度评分（权重 5%）：
   S_latency = max(0, 1 - 响应时间 / 超时时间)
   权重故意设很低，保证人工 Miner 有竞争力

D. 三振出局（Strike System）：
   追踪每个 Miner 最近 10 次探针的判断对错记录（滚动窗口）：
   - 0-2 次判错 → 正常参与
   - 3 次判错 → 黄牌警告：共识投票权重降为 0，只能继续做探针赎回信用
   - 4 次判错 → 红牌封禁：权重归零，冷却 3 个 epoch 后可恢复
   - 5 次以上 → 永久除名：踢出网络，需重新注册（烧 TAO）
   这个机制比单纯降权重更有威慑力——投机者即使偶尔猜对，只要在 10 次内错 3 次就出局。

总分 = 0.60 × 探针均分 + 0.35 × 共识均分 + 0.05 × 速度分
（前提：Strike 未触发封禁，否则总分直接归零）

第 6 页：探针任务生成流程

详细步骤：
1. 从图片库选择一张干净的产品图
2. 随机选择一种篡改方法（Copy-Move / Splicing / 压缩不一致 / 噪声注入）
3. 在图片上执行篡改操作
4. 记录 ground truth：篡改区域的二值 mask + 使用的方法
5. 将篡改图和 ground truth 打包为探针任务
6. 混入任务池（Miner 无法区分探针和真实任务）
7. 当网络准确率连续 3 个 epoch 超过 90% → 自动提升难度（缩小篡改区域、使用更高级的篡改方法）

6. Validator 串通（给特定 Miner 泄题）→ 多 Validator 交叉验证 + Yuma 共识
7. 低质量 Miner 长期占位（持续表现差但不退出）→ 三振出局 Strike System

第 8 页：反作弊机制详解

逐条展开每种防御方法的具体工作原理，每条用一个独立的卡片或区块，标题加粗：

防御 1：加权投票
共识投票不按人头计数，而是按每个 Miner 的历史探针准确率加权。攻击者控制 5 个 Miner 但探针分都很低，投票影响力加起来不如 1 个高分 Miner。Bittensor 注册需烧约 2500 TAO，大规模注册成本极高。60% 的评分权重来自探针（客观评测），Sybil 身份在客观测试上无法获得优势。

防御 2：Commit-Reveal 协议
Phase 1（承诺）：所有 Miner 先提交 hash(结果 + 随机数)，这是不可逆的密码学承诺，提交后无法修改。Phase 2（揭示）：所有 hash 收齐后才开放揭示窗口，Miner 公布原始结果和随机数。Validator 验证 hash 一致性——不匹配直接零分，超时未提交也零分。承诺后、揭示前，任何人只能看到不可逆的 hash 值，无法反推 verdict 或 confidence。

防御 3-4：动态真假比例
探针中真图与假图的比例由 Validator 每个 epoch 动态调整（例如某轮 70% 真 30% 假，下轮反过来），比例不公开。总说"假"的 Miner 在真图探针上全部触发门控（P_correct=0），长期准确率低于 50%。总说"真"同理。Validator 还可主动提高假图比例来加速识别投机者。这类 Miner 因历史探针分低，共识投票权重也趋近于零。

防御 5：方法多样性 + 难度自适应
Forge Engine 随机使用 4 种篡改方法（Copy-Move / Splicing / 压缩不一致 / 噪声注入），每轮随机组合，无法预测。网络准确率连续 3 个 epoch 超过 90% 后自动升级：缩小篡改区域、引入 AI inpainting、混合多种方法、添加后处理。35% 的共识权重奖励真实商家图片上的表现——过拟合探针但真实图表现差的 Miner 会被惩罚。

防御 6：多 Validator 交叉验证
网络有多个独立 Validator，各自生成探针并独立评分。Validator A 串通 Miner X 时，X 只在 A 的探针上得高分，在 B、C 的探针上表现正常。Yuma Consensus 检测到 A 的权重分布与其他 Validator 显著偏离，自动降低 A 的影响力。串通者自身的 emission 也会因异常权重被削减。

防御 7：三振出局 Strike System
追踪每个 Miner 最近 10 次探针的判断对错记录（滚动窗口）。0-2 次判错正常参与；3 次判错黄牌警告（共识权重降为 0，只能做探针赎回信用）；4 次判错红牌封禁（权重归零，冷却 3 个 epoch）；5 次以上永久除名（踢出网络，需重新注册烧 TAO）。这个机制确保低质量 Miner 被快速淘汰，不会长期占位消耗网络资源。比单纯降权重更有威慑力——降权重是"慢慢饿死"，三振出局是"直接开除"。

第 10 页：商业闭环流程

完整业务链条：

1. 商家在电商平台收到退款申请 + 可疑图片
2. 商家通过 HARM API 提交图片
3. Validator 将图片混入任务池（与探针一起分发）
4. 多个 Miner 独立检测（ML + 人工）
5. Commit-Reveal 收集结果
6. Validator 评分：探针评 ground truth、真实任务评共识
7. 生成鉴定报告返回商家：真/假 + 置信度 + 可疑区域
8. 商家据此决定是否同意退款
9. Bittensor 链分配 Emission：41% 给 Miner + 41% 给 Validator + 18% 给子网

变现路径：
Phase 1（1-3 月）：纯合成探针，建立 Miner 生态
Phase 2（4-6 月）：接入商家 API，按查询收费
Phase 3（6 月+）：扩展到保险理赔、法律取证、跨境电商
