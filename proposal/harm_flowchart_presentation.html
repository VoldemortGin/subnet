HARM · Validator

Validator Workflow

Validator creates an indistinguishable mixture of real merchant queries and self-generated probe tasks, then turns Miner performance into Bittensor weights.

02Validator Pipeline

1Image Source Management

Task pool is composed of 70% real merchant queries and 30% probe tasks created by the Validator.

2Forge Engine

Generate fake images from clean originals and record the hidden answer key.

Copy-MoveSplicingCompression MismatchNoise Injection

Each probe stores tampering method + ground truth mask.

3Task Distribution

Mix probes and real tasks, then send the same image to all active Miners.

4Collect Responses

Receive committed/revealed Miner outputs: verdict, confidence, mask, method.

5Scoring

Probes are scored against ground truth. Real tasks are scored by weighted vote.

6Submit Weights

Convert Miner scores into subnet weights and submit them to the Bittensor chain.

HARM · Anti-Copying

Commit-Reveal Anti-Copying Protocol

Miners commit to an irreversible hash before anyone can reveal their actual answer, preventing free-riding and copied responses.

04Commit → Reveal
T0

Distribute Image

Validator sends the same image to all Miners.

T1

Phase 1 · Commit

Each Miner submits hash(result + nonce).

T2

Commit Window Closes

All Miner hashes are collected. Results remain hidden.

T3

Phase 2 · Reveal

Miners reveal original result and nonce.

T4

Verify + Score

Hash mismatch or timeout → zero score. Valid reveals enter scoring.

Key point: Between commit and reveal, Miners only see irreversible hash values from others. They cannot reverse-engineer verdict, confidence, or mask, so copying is structurally blocked.
HARM · Scoring

Scoring Pipeline

Objective probes dominate the score, real merchant images use weighted consensus, and the Strike System turns repeated guessing into immediate exclusion.

05Weights + Strikes

A · Probe Task Score 60%

Ground truth available.

Verdict correctContinue scoring
→
Verdict wrongScore = 0 + one Strike
  • Confidence Calibration: 70% · C_cal = 1 - |confidence - ideal_value|
  • Tampered Region IoU: 20% optional bonus
  • Method Identification: 10% optional bonus

B · Consensus Task Score 35%

No ground truth. Weighted majority vote where each Miner's vote weight equals historical probe accuracy.

Agree with weighted majority → 1 point · Disagree → 0 points

C · Latency Score 5%

Low weight keeps human Miners competitive.

S_latency = max(0, 1 - response_time / timeout)

Total Score

0.60 × avg(Probe) + 0.35 × avg(Consensus) + 0.05 × Latency

Prerequisite: Strike not triggered; otherwise total score = 0.

D · Strike System

Track each Miner's last 10 probe results in a rolling window.

0-2 wrongNormal: full participation.
3 wrongYellow Card: consensus voting weight drops to 0; probes only until credit rebuilds.
4 wrongRed Card: weight set to 0; frozen for 3 epochs.
5+ wrongPermanent Ban: kicked from network; must re-register and burn TAO.
More deterrent than slow weight reduction: speculators who guess wrong 3 times in 10 are immediately cut off.

HARM · Defense Detail

Anti-Gaming Detailed Explanations

The system combines objective probe evaluation, hidden ratios, commitment cryptography, adaptive forge difficulty, and multi-Validator checks.

08Seven Defenses

Defense 1 · Weighted Voting

Consensus voting is weighted by historical probe accuracy, not identity count. Five weak Sybil Miners have less influence than one strong Miner. Bittensor registration burns ~2500 TAO, making large-scale Sybil attacks expensive. Probe tasks contribute 60% of total score, where Sybil identities gain no advantage.

Defense 2 · Commit-Reveal Protocol

Phase 1: Miners submit hash(result + nonce), an irreversible commitment. Phase 2: after all hashes are collected, Miners reveal result and nonce. Mismatch or timeout results in zero score. Between phases, only hash values are visible, so verdict and confidence cannot be copied.

Defense 3-4 · Dynamic Real/Fake Ratio

The Validator changes authentic/tampered probe ratios each epoch and never discloses them. Always-fake fails on authentic probes; always-real fails on tampered probes. Validators can increase fake ratio to flush out speculators. Low probe scores drive consensus voting weight toward zero.

Defense 5 · Method Diversity + Difficulty Adaptation

The Forge Engine randomly selects from Copy-Move, Splicing, Compression Mismatch, and Noise Injection. Once network accuracy exceeds 90% for 3 epochs, difficulty escalates with smaller regions, inpainting, combined methods, and post-processing. Consensus rewards real-image performance, penalizing overfit Miners.

Defense 6 · Multi-Validator Cross-Verification

Multiple Validators generate and score probes independently. If Validator A colludes with Miner X, X only looks good on A's probes but not on B's or C's. Yuma Consensus detects anomalous weight distributions and reduces A's influence and emission.

Defense 7 · Strike System

Last 10 probe verdicts are tracked. 3 wrong triggers Yellow Card; 4 wrong triggers Red Card and 3-epoch freeze; 5+ wrong triggers permanent ban and re-registration. This eliminates low-quality Miners quickly instead of slowly starving them by weight reduction.
