Page 04 · Anti-Copy Protocol

Commit-Reveal 防抄袭协议

核心思想：先承诺答案的哈希，再公开答案。Miner 在承诺后、揭示前看不到别人的原始答案，因此无法复制。
4 / 9
T0
任务下发
Validator 将同一张图片下发给所有 Miner。
T1
Phase 1：承诺
每个 Miner 独立计算，提交 hash(结果 + 随机数)。
T2
Hash 收齐
所有承诺到齐后，进入揭示阶段。
T3
Phase 2：揭示
Miner 公布原始结果 + 随机数。
T4
验证与评分
Validator 验证 hash 一致性，不一致者直接判零分，再进入评分。
关键点提交 hash 之后、揭示之前，其他 Miner 只能看到不可逆的承诺值，无法看到 verdict、confidence、mask 或 method，因此无法搭便车抄袭答案。
Page 05 · Scoring

评分流程

评分由探针任务、共识任务和速度三部分组成。速度权重故意很低，避免把人工 Miner 排除在竞争之外。
5 / 9
A. 探针任务评分
60%
有 ground truth，是最核心的可靠性信号。
门控：verdict 判断正确 → 继续评分；判断错误 → 直接 0 分。
置信度校准 70%：C_cal = 1 - |confidence - 理想值|
篡改区域 IoU 20%：预测 mask 与真实 mask 的交并比，可选加分。
方法识别 10%：正确识别篡改方法得 1 分，可选加分。
B. 共识任务评分
35%
无 ground truth，用网络共识评估真实商家任务。
每个 Miner 的投票权重 = 其历史探针准确率。
与加权多数一致 → 得 1 分。
与加权多数不一致 → 得 0 分。
C. 速度评分
5%
低权重：鼓励效率，但不牺牲人工 Miner 的竞争力。
S_latency = max(0, 1 - 响应时间 / 超时时间)
越快越好，但不会成为主要收益来源。
总分 = 0.60 × 探针均分 + 0.35 × 共识均分 + 0.05 × 速度分

反作弊机制总览

6 种攻击方式，6 种对策。每种攻击都有针对性的协议级或经济级防御。
7 / 10
1. 女巫攻击一人注册多个 Miner 操纵投票
→
加权投票 + 经济门槛投票权重 = 探针准确率，不按人头数；注册烧 TAO
2. 搭便车/抄袭等别人答完再复制答案
→
Commit-Reveal 协议先提交答案哈希，全部收齐后才揭示，无法看到别人的答案
3. 总说”假”永远回答”篡改”赌概率
→
动态真假比例 + 门控探针真假比例每轮变化且不公开；判错触发门控直接零分
4. 总说”真”永远回答”未篡改”赌概率
→
同上机制完全对称；历史探针分低则共识投票权重趋近于零
5. 模型过拟合专门针对 Validator 造假模式优化
→
方法多样性 + 难度自适应4 种篡改方法随机组合；准确率超 90% 自动升级难度
6. Validator 串通提前泄露探针答案给特定 Miner
→
多 Validator 交叉验证Yuma 共识检测异常权重分布，自动降低串通者影响力
Page 08 · Anti-Fraud Details

反作弊机制详解

每种防御方法的具体工作原理。
8 / 10
防御 1：加权投票
→
共识投票不按人头计数，而是按每个 Miner 的历史探针准确率加权。假设攻击者控制 5 个 Miner，但这 5 个在探针上都表现差，那它们的投票影响力加起来也不如 1 个高分 Miner。同时 Bittensor 注册需烧约 2500 TAO，大规模注册的经济成本极高。60% 的评分权重来自探针（客观评测），Sybil 身份在客观测试上无法作弊。
防御 2：Commit-Reveal
→
Phase 1（承诺）：所有 Miner 先提交 hash(结果 + 随机数)，这是不可逆的密码学承诺，提交后无法修改。Phase 2（揭示）：所有 hash 收齐后才开放揭示窗口，Miner 公布原始结果和随机数。Validator 验证 hash 一致性——不匹配直接零分，超时未提交也零分。Miner 在承诺后、揭示前只能看到不可逆的 hash 值，无法反推任何人的 verdict 或 confidence。
防御 3-4：动态比例
→
探针中真图与假图的比例由 Validator 每个 epoch 动态调整（例如某轮 70% 真图 30% 假图，下轮反过来），比例不公开。总说”假”的 Miner 在真图探针上全部触发门控（P_correct=0），长期准确率被拉到 50% 以下。总说”真”同理。Validator 还可主动提高假图比例来加速识别投机者。这类 Miner 因历史探针分低，共识投票权重也趋近于零。
防御 5：方法多样性
→
Forge Engine 随机使用 4 种篡改方法（Copy-Move / Splicing / 压缩不一致 / 噪声注入），每轮随机组合。当网络准确率连续 3 个 epoch 超过 90%，自动升级：缩小篡改区域、引入 AI inpainting、混合多种方法、添加后处理。35% 的共识权重奖励真实商家图片上的表现——过拟合探针但真实图表现差的 Miner 会在这 35% 上被惩罚。
防御 6：交叉验证
→
网络有多个独立 Validator，各自生成探针并独立评分。如果 Validator A 串通 Miner X，X 只在 A 的探针上得高分，在 B、C 的探针上表现正常。Yuma Consensus 检测到 A 的权重分布与其他 Validator 显著偏离，自动降低 A 的影响力。epoch 后 Validator 可交叉审计探针 ground truth。串通者自身的 emission 也会因异常权重被削减。
