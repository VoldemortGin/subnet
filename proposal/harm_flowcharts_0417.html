Commit-Reveal 防抄袭协议

所有 Miner 先提交不可逆 hash 承诺，等承诺全部收齐后才揭示原始结果，从时间结构上切断“看别人答案再抄”的路径。
Phase 1 承诺 · Phase 2 揭示
T0
Validator 下发图片
同一任务发给多个 Miner。
T1
提交 hash
Miner 独立计算，提交 hash(结果 + 随机数)。
Phase 1：承诺阶段
T2
hash 收齐
所有人只能看到不可逆的 hash。
T3
揭示原始结果
公布 verdict、confidence、mask、method 与随机数。
Phase 2：揭示阶段
T4
验证并评分
hash 不一致者直接零分；一致者进入评分。
为什么不能抄？

承诺后、揭示前，别人只看到 hash，无法反推出 verdict 或 confidence。

为什么不能改？

揭示结果必须与先前 hash 完全一致；随机数也必须匹配。

违规怎么处理？

hash 不一致、超时未揭示、格式错误，均可直接判零分。

commit = hash(verdict, confidence, mask, method, nonce)   →   reveal = result + nonce   →   verify(commit)

评分流程

总分由探针任务、共识任务与速度评分组成；如果 Strike System 触发封禁，总分直接归零。
60% 探针 · 35% 共识 · 5% 速度
A · 60%
探针任务评分
先过门控：verdict 正确才继续；错误直接 0 分并记 1 次 Strike。
  • 置信度校准 70%
  • mask IoU 20%
  • 方法识别 10%
B · 35%
共识任务评分
真实任务无 ground truth，因此用加权多数投票；投票权重等于历史探针准确率。
C · 5%
速度评分
S_latency = max(0, 1 - 响应时间 / 超时时间)。权重很低，保证人工 Miner 仍有竞争力。
D · 三振出局 Strike System：最近 10 次探针滚动窗口
0-2 次判错正常参与，继续获得探针分与共识权重。
3 次判错黄牌警告：共识投票权重降为 0，只能做探针赎回信用。
4 次判错红牌封禁：权重归零，冷却 3 个 epoch 后可恢复。
5 次以上永久除名：踢出网络，需重新注册并烧 TAO。
威慑效果投机者即使偶尔猜对，只要 10 次内错 3 次就出局。
总分 = 0.60 × 探针均分 + 0.35 × 共识均分 + 0.05 × 速度分
前提：Strike 未触发封禁；若触发封禁，总分直接归零。

反作弊机制详解

把总览页里的防御机制展开成可执行的规则：谁有投票权、何时揭示、如何动态调整、怎么处理串通和低质量节点。
7 层防御
防御 1：加权投票

共识不按人头计数，而按 Miner 历史探针准确率加权。攻击者控制多个低分 Miner，影响力也不如一个高分 Miner。注册需烧 TAO，批量注册成本高；60% 分数来自客观探针，Sybil 身份本身没有优势。

防御 2：Commit-Reveal 协议

Phase 1 先提交 hash(结果 + 随机数)，提交后无法修改；Phase 2 在所有 hash 收齐后揭示原始结果。Validator 验证 hash 一致性，不匹配或超时直接零分。

防御 3-4：动态真假比例

探针中真图与假图比例每个 epoch 动态调整且不公开。总说“假”会在真图探针上触发门控，总说“真”同理。Validator 可主动调高反向比例，加速识别投机者。

防御 5：方法多样性 + 难度自适应

Forge Engine 随机使用 Copy-Move、Splicing、压缩不一致、噪声注入，并可升级到 inpainting、混合篡改和后处理。共识任务奖励真实商家图片表现，抑制只会刷探针的模型。

防御 6：多 Validator 交叉验证

多个独立 Validator 各自生成探针并独立评分。若 Validator A 串通 Miner X，X 只在 A 上异常高分；Yuma Consensus 可检测权重分布偏离并降低 A 的影响力。

防御 7：三振出局 Strike System

追踪最近 10 次探针判断。0-2 错正常；3 错黄牌，投票权重归零；4 错红牌封禁，冷却 3 epoch；5 错以上永久除名，需重新注册烧 TAO。

核心思想：用客观探针控制质量，用密码学承诺防抄袭，用经济门槛和滚动淘汰提高作恶成本。
