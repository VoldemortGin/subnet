Page 04 · Commit-Reveal

Commit-Reveal 防抄袭协议

先提交不可逆 hash，再统一揭示答案。这样 Miner 在承诺之后、揭示之前无法看到其他人的结果，也无法临时改答案。
Phase 1 承诺 · Phase 2 揭示
T0
任务下发
Validator 将同一批图片发给所有 Miner。
T1
提交承诺
每个 Miner 独立计算，提交 hash(结果 + 随机数)。
Phase 1
T2
Hash 收齐
所有 Miner 的承诺值到齐后，系统关闭承诺窗口。
T3
揭示答案
Miner 公布原始结果 + 随机数。
Phase 2
T4
验证与评分
Validator 重新计算 hash；不一致者或超时者直接零分。
不可逆
只看到 hash 值，无法反推出 verdict、confidence 或 mask。
不可篡改
提交 hash 之后再改结果，会导致揭示阶段验证不通过。
防搭便车
答案公开前必须先承诺，无法等别人答完再复制。
Page 05 · Scoring

评分流程

评分由探针任务、共识任务和速度三部分组成。重点放在客观探针评分，速度只占 5%，避免把人工 Miner 排除在外。
60% + 35% + 5%
A. 探针任务评分 · 60%
有 ground truth。先做门控：verdict 判断正确才继续评分；判断错误直接 0 分。
置信度校准 70%IoU 20%方法识别 10%
C_cal = 1 - |confidence - 理想值|
B. 共识任务评分 · 35%
无 ground truth。使用加权多数投票：每个 Miner 的投票权重 = 其历史探针准确率。与加权多数一致得 1 分，不一致得 0 分。
C. 速度评分 · 5%
响应越快分越高，但权重故意设低，保证人工 Miner 仍有竞争力。
S_latency = max(0, 1 - 响应时间 / 超时时间)
总分公式
总分 = 0.60 × 探针均分
     + 0.35 × 共识均分
     + 0.05 × 速度分
设计意图：客观测试决定主权重；真实任务共识验证商业表现；速度只做轻量激励。
客观性优先抗投机兼容人工

Page 08 · Anti-Cheat Details

反作弊机制详解

这一页把每种防御的工作原理展开：不是单点防御，而是把客观探针、密码学承诺、经济成本和多 Validator 共识组合起来。
组合式防御
防御 1：加权投票
共识投票不按人头计数，而是按每个 Miner 的历史探针准确率加权。攻击者控制多个低分 Miner，影响力也不如一个高分 Miner。注册需要经济成本，且 60% 评分权重来自客观探针，Sybil 身份无法凭数量获得优势。
防御 2：Commit-Reveal 协议
Phase 1 先提交 hash(结果 + 随机数)，提交后无法修改。Phase 2 等所有 hash 收齐后才揭示原始结果和随机数。Validator 校验 hash，不匹配或超时直接零分；承诺后、揭示前，其他人只能看到不可逆 hash。
防御 3-4：动态真假比例
探针中真图与假图比例每个 epoch 动态调整且不公开。总说“假”的 Miner 会在真图探针上触发门控，总说“真”同理。Validator 还可主动提高某类样本比例来加速识别投机者。
防御 5：方法多样性 + 难度自适应
Forge Engine 随机使用多种篡改方式并可组合。连续 3 个 epoch 网络准确率超过 90% 后升级难度：缩小区域、引入 AI inpainting、混合多方法、添加后处理。真实商家任务共识也会惩罚只会探针的过拟合 Miner。
防御 6：多 Validator 交叉验证
多个独立 Validator 各自生成探针并独立评分。若 Validator A 与 Miner X 串通，X 只会在 A 的探针中异常高分，在 B、C 的探针上表现正常。Yuma Consensus 可识别 A 的权重分布与其他 Validator 显著偏离，并降低其影响力，串通者自身 emission 也会受损。

Page 10 · Business Loop

商业闭环流程

HARM 的业务闭环连接商家退款场景、去中心化检测网络、鉴定报告和链上激励，最终形成可收费的 API 服务。
检测服务 → 报告 → Emission
1. 退款申请
商家收到退款申请与可疑图片。
2. API 提交
商家通过 HARM API 上传图片。
3. 混入任务池
Validator 与探针一起分发。
4. Miner 检测
ML + 人工多路独立判断。
5. Commit-Reveal
收集承诺与揭示结果。
6. Validator 评分
探针评 ground truth，真实评共识。
7. 返回报告
真/假 + 置信度 + 可疑区域。
8. 商家决策
决定是否同意退款。
9. 链上分配
41% Miner + 41% Validator + 18% 子网。
Phase 1 · 1-3 月
纯合成探针，建立 Miner 生态，先把检测网络与评分机制跑起来。
Phase 2 · 4-6 月
接入商家 API，按查询收费，把真实退款场景引入网络。
Phase 3 · 6 月+
扩展到保险理赔、法律取证、跨境电商等更高价值场景。
